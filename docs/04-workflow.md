# 04 — WORKFLOW (มุมคนใช้งาน)

## ก่อนประชุม
- ไม่ต้องทำอะไร — แอป Django รันเป็น server รออยู่แล้ว (ไม่ต้องเปิดตอนประชุม)

## ระหว่างประชุม
1. กด Record ใน Teams (หรืออัดมือถือวางบนโต๊ะ)

## หลังประชุม
2. ไฟล์เข้าระบบอัตโนมัติ
   - Teams webhook ดึง .mp4 จาก OneDrive  หรือ
   - คน upload ไฟล์เสียงหน้าเว็บ  หรือ
   - วางไฟล์ลงโฟลเดอร์ inbox (watchdog จับ)
3. ระบบแปลง + วิเคราะห์เอง (~2–5 นาที ถ้ามี GPU)
4. ครั้งแรกของแต่ละคน: map SPEAKER_xx → ชื่อจริง (ทำครั้งเดียว ระบบจำเสียง)

## ตอนอยากรู้ข้อมูล
5. เปิด Obsidian พิมพ์ค้น "เครื่อง A" → เห็นประวัติเสีย/ซ่อมทั้งหมด
6. หรือพิมพ์ใน Teams chat: "@bot เครื่อง A" → bot ตอบประวัติล่าสุด

---

## จุดสำคัญ: เก็บเสียง ≠ แปลงเสียง (แยกกัน)

```
งาน A: เก็บเสียง (ตอนประชุม)      งาน B: แปลง+วิเคราะห์ (หลังจบ)
  Teams / มือถือ ทำ          →      แอป Django ทำ (รัน server ตลอด)
```
แอปคุณทำแค่ **งาน B**. ไม่ต้องรันตอนประชุม. ประชุมจบมีไฟล์เข้ามา → มันทำให้เอง

## ได้ไฟล์เสียงมา 3 ทาง
| ทาง | ใครเก็บ | แอปต้องรันตอนประชุม? |
|-----|---------|----------------------|
| 1. Teams record → Graph ดึง .mp4 | Teams | ไม่ต้อง |
| 2. มือถือ/อัดจอ → upload | คน | ไม่ต้อง |
| 3. แอปอัดเอง | แอป | ต้อง (ไม่แนะนำ) |
